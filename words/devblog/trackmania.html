Riffing on TrackMania Logos

  • April 13th, 2021

  • 5 min read

I recently noticed these logos while watching a few videos about the TrackMania series, which I've only been vaguely familiar with over the years. The games are about racing the clock on highly modular racetracks, and I can remember way back in the early days of YouTube when some custom tracks in the earlier entries went viral for their insane geometry. When the driver is in first-person mode, it becomes a sort of digital rollercoaster.

The games, sharing an engine and released within a year of one another, share clean, borderline sterile art directions befitting of most modern racing sims, but these small splashes of color stood out to me. I wanted to see if that sort of palette-blocking effect could be done easily in pure CSS.

basic pure-CSS concept

The easy method necessitates a monospace font type, because of the predictable letter spacing. The actual logos use Arial as the font, which is sans-serif and comes with its own specialized kerning that makes things pretty annoying, with each letter requiring individual padding to unset that kerning. I suppose a monospace Arial lookalike would be the best case scenario, but I'm not all that interested in hunting one down, lol.

For the sake of convenience, here's an approximation for the Nations logo using pure monospace:

N a t i o n s

Here's one way to do it:


    <div id="nations" class="noselect">
        <span class="nations-block" style="background-color: #66a115">N</span>
        <span class="nations-block" style="background-color: #60ba00">a</span>
        <span class="nations-block" style="background-color: #98e13a">t</span>
        <span class="nations-block" style="background-color: #80ca01">i</span>
        <span class="nations-block" style="background-color: #5ab710">o</span>
        <span class="nations-block" style="background-color: #7fdf01">n</span>
        <span class="nations-block" style="background-color: #50d500">s</span>
    </div>

The HTML is a series of span s wrapped up in a standard div with inline styling for the specific coloring. It's a little cluttered but ultimately a lot cleaner than a different CSS class or id for every letter. That .noselect class for the div prevents the user from highlighting the text so the illusion of this being an image isn't broken. You can replicate that behavior with this monstrosity: 


    .noselect {
        -webkit-touch-callout: none; /* iOS Safari */
        -webkit-user-select: none; /* Safari */
        -khtml-user-select: none; /* Konqueror HTML */
        -moz-user-select: none; /* Old versions of Firefox */
        -ms-user-select: none; /* Internet Explorer/Edge */
        user-select: none; /* Chrome, Edge, Opera and Firefox */
    }
    /* we really need to standardize this, don't we? */

Back to the Nations logo, here's what the stylesheet is doing: 


    div#nations {
        display: flex;
    }
    span.nations-block {
        padding: 0 8px; /* creates the space to color in */
        line-height: 26px;
        font-size: 30px; /* larger than line-height for vertical compactness */
        font-family: monospace;
    }

The important takeaway is that this div is held together by the display: flex; option, which packs things in as tightly as possible unless told not to. The reasoning for this choice is that the other display options, including the default in this scenario, try to give span objects some built-in spacing between each other that we don't want.

Trying to circumvent this by placing all of the span s on one line of HTML code removes our ability to alter the line-height variable, which we need to be shorter than the font size to bring the bounds of the coloring down snugly to the lettering. Later, we'll see how to get around using display: flex altogether.

The logo for Trackmania United is more of the same, with a bolder font and some tighter padding:

U n i t e d

I don't like this one as much as Nations - it's a bit more garish. But it is a good excuse to play around with some unusual text patterns, like inline tilting. 


    <div id="united" class="noselect">
        <span class="united-block" style="background-color: #fe0000">U</span>
        <span class="united-block" style="background-color: #ffa900">n</span>
        <span class="united-block" style="background-color: #fdfe01">i</span>
        <span class="united-block" style="background-color: #54ff00">t</span>
        <span class="united-block" style="background-color: #00d3ff">e</span>
        <span class="united-block rotate33" style="background-color: #007eff">d</span>
    </div>
            

    div#united {
        display: flex;
        padding-left: 33%;
    }
    span.united-block {
        padding: 0 9px;
        line-height: 27px;
        font-size: 35px;
        font-family: monospace;
        font-weight: 500;
    }

The "d" has an additional class called .rotate33 which tilts it 33 degrees to the right using the transform element. I am not 100% confident that this is tilt angle but it's certainly the closest common angle to the source. You can achieve this effect like this: 


    .rotate33 {
        -webkit-transform: rotate(33deg); /* Safari */
        -moz-transform: rotate(33deg); /* Old versions of Firefox */
        -ms-transform: rotate(33deg); /* Internet Explorer/Edge */
        -o-transform: rotate(33deg); /* Old versions of Opera */
        transform: rotate(33deg); /* Chrome, Edge, Opera and Firefox */
    }
    /* seriously, we need to standardize this stuff. */

In all, the actual implementation of this color-blocking technique is nothing very difficult to achieve in CSS. The simplicity of the task and interesting composition of the colors, especially similar ones like the greens in Nations , compelled me to play around with them a little.

fixes and design tweaks

While I feel like the idea that TrackMania had was sort of eye-catching, it also felt underdeveloped. I decided to play around with the idea a bit, and I landed on something a bit softer and more square, taking the palette-splatter idea and trying out different colors.

Unfortunately, adding the border and outline that I wanted revealed that display: flex; option wasn't keeping the whole div horizontally "tight," just the span s. The rest of the div still extended on the end of the container, like you see here:

p r o j e c t s
p o r t f o l i o
w o r d s
a b o u t m e

Using display: inline-flex; helps a little, and it's the only difference between the two groups, but multiple words in a row are forced into a row without something between them. In a perfect world, they're a complete object that doesn't get pushed around or snapped into some unwanted position.

If you're viewing this on mobile, try looking at it in landscape mode to see what I'm talking about.

Mercifully, simply hitching these inline-flex objects onto an unordered list can give you the structure you need, so if you wanted to use this concept for navigation bar elements it works great.

Lists are very powerful in HTML and can be organized into plenty of shapes that make designs like this functional. It's not like this sort of text can be used as a font, or to display longform information like a sentence or paragraph. I feel this general idea is an eye-catching way to display small amounts of information like names or titles of various tabs, so a coherent use case in a list or navigation tool of some kind feels like a fitting end. It is from a logo, after all.
